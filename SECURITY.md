## Security Policy

At AnarchyGem, we take the security of our tools and the privacy of our users seriously. We appreciate responsible disclosure of security vulnerabilities. This document outlines the process for reporting security issues and what to expect after reporting.

### Reporting a Vulnerability

If you discover a security vulnerability, please report it to us immediately. Do not disclose the vulnerability publicly until we have had a chance to address it.

**Preferred Contact Method:**

Please send an email to `security@anarchygem.org` (placeholder email) with a detailed description of the vulnerability. If possible, include:

*   A clear description of the vulnerability.
*   Steps to reproduce the vulnerability.
*   The potential impact of the vulnerability.
*   Any proof-of-concept code or exploits.

### What Constitutes a Security Issue?

A security issue is generally defined as a flaw that could lead to:

*   Unauthorized access to data or systems.
*   Data corruption or loss.
*   Denial of service.
*   Bypassing intended security controls.
*   Any other compromise of the integrity, confidentiality, or availability of the software or user data.

**Examples of security issues:** Cross-site scripting (XSS), SQL injection, authentication bypasses, remote code execution, sensitive data exposure.

**What is NOT a security issue (and should be reported as a bug or feature request):**

*   Typographical errors.
*   User interface bugs.
*   Feature requests or suggestions for new functionality.
*   Issues that require physical access to a user's device.
*   Vulnerabilities in third-party websites or services not directly controlled by AnarchyGem.

### Expected Response Timeline

We aim to respond to all security reports within the following timeframes:

*   **Initial Acknowledgment:** Within 2 business days.
*   **Assessment and Status Update:** Within 7 business days, we will provide an initial assessment of the reported vulnerability and an estimated timeline for a fix.
*   **Resolution:** The time to resolve a vulnerability will depend on its complexity and severity. We will keep you informed of our progress.

We kindly request that you give us a reasonable amount of time to fix the vulnerability before any public disclosure. We believe in collaborative security and appreciate your cooperation in making AnarchyGem more secure for everyone.

Thank you for helping to keep AnarchyGem secure and sovereign.
