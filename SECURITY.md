## Security Policy

We're building AnarchyGem to be robust, but no system is perfect. If you find a security vulnerability, we want to know about it so we can fix it.

### Reporting a Vulnerability

**Sensitive Security Issues:**
For vulnerabilities that could pose a significant risk (e.g., data breaches, unauthorized access), please use GitHub's Private Vulnerability Reporting feature. You can find this under the "Security" tab of this repository, then select "Report a vulnerability." This ensures your report remains confidential while we address it.

**Non-Sensitive Bugs & Issues:**
If you've found a bug that isn't a direct security vulnerability (e.g., a functional bug, a UI glitch, or a minor issue that doesn't compromise data or access), please open a standard GitHub issue.

### What Constitutes a Security Issue?

A security issue is a flaw that directly threatens the integrity, confidentiality, or availability of AnarchyGem or its users' data. This includes:

*   Unauthorized access to data or systems.
*   Data corruption or loss.
*   Bypassing intended security controls.
*   Remote code execution.
*   Sensitive data exposure.

**What is NOT a security issue (report these as regular bugs/issues):**

*   Typographical errors.
*   User interface bugs.
*   Feature requests.
*   Issues requiring physical access to a device.
*   Vulnerabilities in third-party services not directly controlled by AnarchyGem.

### Our Commitment

We'll acknowledge your report promptly and work to address valid vulnerabilities. We appreciate your help in keeping AnarchyGem secure.
